# 《客户跟进事项》详细设计（DD）

- 文档版本：v1.0
- 基于文档：`《客户跟进事项》PRD.pdf`（v1.0，2026/03/26）
- 更新时间：2026/03/28
- 适用范围：CRM 模块（客户管理）

## 1. 设计目标

在不改变 CRM 现有主流程的前提下，构建一套“可记录、可追溯、可筛选、可管理”的客户跟进事项能力，满足以下目标：

1. 客户触达行为结构化沉淀
2. 跟进过程可回放、可筛选、可编辑
3. 待跟进任务可识别、可闭环（已跟进/撤销已跟进）
4. 为 KPI 与后续 BI 分析提供可用数据

## 2. 页面与信息架构

### 2.1 页面层级（现版实现）

1. 客户列表页（主页面）
2. 客户详情弹窗（由客户列表页打开）
3. 新增跟进弹窗（可从列表/详情/历史进入）
4. 跟进历史弹窗（可从列表/详情进入）
5. 跟进记录详情弹窗（由历史卡片进入）
6. 删除确认弹窗（由详情弹窗进入）

说明：`customer-detail.html` 已作为兼容入口，自动跳转到 `customer-list.html?openDetail=1`。

### 2.2 关键入口

1. 客户列表 → 查看详情（打开客户详情弹窗）
2. 客户列表 → 新增跟进（打开新增弹窗）
3. 客户列表 → 跟进历史（打开历史弹窗）
4. 客户详情弹窗 → 新增跟进
5. 客户详情弹窗 → 跟进历史
6. 客户详情弹窗“下次待跟进数”卡片 → 历史弹窗（自动应用待跟进筛选）
7. 历史卡片 → 记录详情

## 3. 功能详细设计

### 3.1 客户列表页

#### 展示内容

1. 客户基础列：名称、行业、阶段、负责人
2. 最新跟进进度：跟进类型标签 + 最新时间 + 内容摘要
3. 操作列：新增跟进、跟进历史、查看详情

#### 交互规则

1. 支持客户名称关键词筛选
2. 支持客户阶段筛选
3. 支持“重置筛选”
4. 支持顶部“重置 Mock 数据”

### 3.2 客户详情弹窗

#### 模块结构

1. 客户基础信息卡
2. 跟进记录模块
3. 模块内统计卡（累计跟进次数、最近7天跟进次数、下次待跟进数）
4. 最近一条跟进记录摘要

#### 关键规则

1. 点击“下次待跟进数”卡片：打开历史弹窗并自动筛选“下次跟进时间 > 当前时间 且 未标记已跟进”
2. 最近记录摘要点击进入详情弹窗（不强制先打开历史弹窗）
3. 若跟进内容为空，统一展示：`跟进内容：待补充`

### 3.3 新增跟进弹窗

#### 字段定义

1. 跟进类型（必填）
2. 自定义标签名称（当类型为“自定义标签”时必填）
3. 跟进时间（必填）
4. 跟进人（默认当前用户，展示姓名+工号）
5. 跟进内容（选填）
6. 下次跟进时间（选填）

#### 校验规则

1. 跟进类型不能为空
2. 跟进时间不能为空
3. 若选择自定义标签，则名称不能为空
4. 跟进内容可为空

#### 提交结果

1. 写入新记录（默认 `next_followup_done = false`）
2. 刷新列表、详情、历史数据
3. 更新统计卡
4. 记录埋点 `followup_submit_success`

### 3.4 跟进历史弹窗

#### 展示结构

1. 筛选区：类型、开始日期、结束日期、重置筛选
2. 时间线卡片区：按跟进时间倒序
3. 底部栏：共 X 条 + 新增跟进按钮

#### 体验优化

1. 日期输入框右侧 `×` 清空按钮（有值显示，无值隐藏）
2. 滚动到底自动加载更多（无需点击“查看更多”）
3. 若卡片无下次跟进时间，不展示该字段
4. 若卡片跟进内容为空，展示 `跟进内容：待补充`

### 3.5 记录详情弹窗

#### 展示字段

1. 跟进类型
2. 跟进时间
3. 跟进人（姓名+工号）
4. 下次跟进时间（有值显示，且支持“已跟进状态”）
5. 跟进内容（为空显示 `跟进内容：待补充`）
6. 创建时间 / 更新时间

#### 操作能力

1. 编辑
2. 删除（含二次确认）
3. 已跟进状态切换（可逆）

#### 已跟进可逆规则

1. 当存在下次跟进时间时显示按钮
2. 未跟进：按钮文案 `标记已跟进`
3. 已跟进：按钮文案 `撤销已跟进`
4. 点击后更新 `next_followup_done` 并刷新统计与筛选结果
5. 日期后缀展示 `（已跟进）`

## 4. 数据模型设计

## 4.1 客户对象

```json
{
  "id": "cus_001",
  "name": "上海星澜智能科技有限公司",
  "industry": "企业服务 / AI SaaS",
  "stage": "方案沟通",
  "owner": "李明",
  "phone": "138****8888"
}
```

## 4.2 跟进记录对象

```json
{
  "id": "fu_1001",
  "typeKey": "电话沟通",
  "typeLabel": "电话沟通",
  "followupTime": "2026-03-25T15:20",
  "person": "李明",
  "content": "...",
  "nextFollowupTime": "2026-03-29T10:00",
  "nextFollowupDone": false,
  "createdAt": "2026-03-25T15:20",
  "updatedAt": "2026-03-25T15:20"
}
```

### 4.3 衍生指标口径

1. 累计跟进次数：记录总数
2. 最近7天跟进次数：`followupTime` 落在 `[now-7d, now]`
3. 下次待跟进数：`nextFollowupTime > now AND nextFollowupDone = false`

## 5. 状态与流程

### 5.1 跟进状态流

1. 新增记录（可有/无下次跟进时间）
2. 若有下次跟进时间，默认 `未跟进`
3. 在详情中可“标记已跟进”
4. 已跟进状态可“撤销已跟进”

### 5.2 关闭行为

所有弹窗统一右上角 `X` 关闭；子弹窗关闭后保留父弹窗上下文。

## 6.1 事件清单

1. `followup_create_click`
2. `followup_form_view`
3. `followup_form_fill_progress`
4. `followup_submit_success`
5. `followup_history_detail_click`
6. `followup_edit`
7. `followup_delete`
8. `followup_filter_reset`
9. `followup_mark_done`
10. `mock_data_reset`

### 6.2 建议公共属性

1. `user_id`
2. `customer_id`
3. `tenant_id`
4. `user_role`
5. `page_name`
6. `event_time`

## 7. 边界与异常处理

1. 无记录时展示空状态并提供新增入口
2. 筛选无结果时展示空状态
3. 删除后若当前详情记录不存在，关闭详情并刷新列表
4. 日期筛选清空后立即恢复默认筛选
5. 跟进内容为空时统一展示 `跟进内容：待补充`

## 8. 验收标准（UAT）

1. 客户详情通过弹窗展示，不再跳转独立页面
2. 新增记录后，列表/详情/历史数据同步更新
3. 下次待跟进数点击后仅展示待跟进事项
4. 记录可“标记已跟进/撤销已跟进”，统计实时联动
5. 日期筛选支持 `×` 快速清空
6. 历史列表滚动到底可自动继续加载
7. 空内容展示为 `跟进内容：待补充`
8. 跟进人统一显示“姓名（工号：xxxxxx）”
