# CRM 客户跟进事项原型（笔试项目说明）

## 1. 项目定位
本项目是围绕「客户跟进事项」需求完成的前端交互原型，核心目标是：
- 在客户列表与客户详情链路中，支持持续录入跟进事件。
- 支持按时间与类型追溯历史记录。
- 支持编辑、删除、筛选、校验与埋点，保证可演示与可复盘。

本原型使用原生 `HTML + CSS + JavaScript` 实现，全部为前端 mock 数据，不依赖后端接口。

## 2. 评审范围说明（重要）
### 本次 PRD 主评审范围
- 客户列表
- 客户详情（弹窗）
- 新增跟进（弹窗）
- 跟进历史（弹窗）
- 跟进详情（弹窗）

### 非本次 PRD 主范围
- 「管理者数据看板（前瞻）」为额外探索模块，不作为本 PRD 核心验收项。
- 如仅按 PRD 评分，可忽略该模块。

## 3. 如何运行
1. 打开项目根目录下 `index.html`（会自动进入客户列表页）。
2. 推荐使用浏览器直接打开，或使用本地静态服务打开均可。
3. 所有交互均可在前端直接演示（无需启动后端）。

## 4. 建议评审路径（5~10 分钟）
1. 进入客户列表页，查看每个客户的最近跟进摘要、最近跟进时间、快捷操作。
2. 在列表页点击「查看详情」，打开客户详情弹窗。
3. 在客户详情的跟进模块点击「新增跟进」，验证表单校验、提交成功、列表与摘要同步。
4. 点击「跟进历史」，验证：
   - 类型筛选
   - 日期筛选
   - 日期清空（输入框右侧 x）
   - 重置筛选
   - 滚动自动加载
5. 点击任意历史卡片进入跟进详情，验证：
   - 编辑（含跟进时间可编辑）
   - 删除二次确认
   - 下次跟进“已跟进/撤销已跟进”可逆
   - 数据联动（待跟进计数自动变化）
6. 返回列表，点击「重置 mock 数据」，验证全局恢复初始状态。

## 5. 功能实现对照（核心）
| 模块 | 已实现内容 | 说明 |
|---|---|---|
| 客户列表 | 客户基础信息、最近跟进摘要、最近跟进时间、查看详情、跟进历史、新增跟进 | 作为业务起点页 |
| 客户详情（弹窗） | 客户基础信息、最近跟进卡、跟进统计卡、新增跟进、跟进历史入口 | 关闭仅影响当前层弹窗 |
| 新增跟进（弹窗） | 跟进类型、跟进时间、跟进人、内容（非必填）、下次跟进时间；实时校验与提交 | 提交后多视图同步 |
| 跟进历史（弹窗） | 时间线卡片、类型筛选、时间筛选、重置筛选、自动滚动加载、总条数显示 | 空状态可引导新增 |
| 跟进详情（弹窗） | 查看完整字段、编辑、删除、已跟进状态切换 | 未设置下次跟进时间时不展示该字段 |
| 埋点与调试 | 关键行为埋点、日志面板开关、事件时间记录 | 跟进内容埋点改为失焦触发（blur） |
| 数据管理 | 本地 mock 数据集中管理、重置按钮恢复初始数据 | 便于评审反复演示 |

## 6. 我在产品上的思考
### 6.1 为什么从“客户列表”开始
真实销售工作流通常从客户池出发，而非先进入单个客户详情。将“新增跟进/跟进历史/查看详情”入口前置在列表页，可降低操作路径长度，提升记录频率。

### 6.2 为什么统一采用弹窗层级
该需求高频“查看-编辑-回看”，如果频繁页面跳转会丢失上下文。采用弹窗叠层可以：
- 保留用户当前定位与筛选上下文。
- 缩短回退路径，提高演示与实操效率。
- 更贴近 B 端系统中台操作习惯。

### 6.3 为什么强调“下次待跟进”
该指标直接关联销售节奏管理。系统支持：
- 自动统计“当前时间之后且未标记已跟进”的事项。
- 在详情中可手动“已跟进/撤销已跟进”做可逆操作。
- 指标随行为实时联动，避免管理口径与一线动作脱节。

### 6.4 为什么埋点要可演示
面试场景下，除了界面功能，还需要体现“可度量”。我增加了埋点开关和实时日志，便于说明：
- 哪些事件可用于漏斗分析（查看、新增、提交、编辑、删除、筛选）。
- 哪些字段行为可用于评估录入质量（如内容字段 blur 时机）。

## 7. 技术与可维护性说明
- 页面按模块拆分在 `pages/` 下，HTML/CSS/JS 分文件组织，方便持续迭代。
- mock 数据集中在前端 store，支持全局状态同步与重置。
- 关键交互（弹窗、筛选、校验、删除确认、toast）做了统一风格和状态反馈。

## 8. 可验证的验收要点（建议给考核员）
- 是否能从列表直接完成全链路（不依赖额外页面跳转）。
- 新增/编辑/删除后，列表摘要、详情、历史是否同步。
- 筛选是否真实生效，是否支持恢复默认。
- 弹窗关闭是否仅关闭当前层，不误关父层。
- 边界值处理是否合理：
  - 跟进内容为空显示“跟进内容：待补充”
  - 下次跟进时间未设置时不冗余展示
  - 已跟进状态支持可逆
